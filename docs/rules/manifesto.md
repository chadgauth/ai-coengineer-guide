# Engineering Manifesto

This document outlines the architectural and strategic goals for the `ai-copilot-guide` repository.

## Project Objectives

1. **Practical Application**: Provide highly technical, discrete examples of how to build AI-driven workflows.
2. **Standardization**: Establish a canonical method for synchronizing AI context across disparate tools (Cursor, Claude Desktop, GitHub Copilot).
3. **High-Fidelity Context**: Demonstrate how passing career goals, specific team dynamics, and structural constraints directly to the AI yields substantially higher-quality code generation.

## Formatting Standards

- **Clarity and Precision**: Explanations must be direct, avoiding unnecessary fluff.
- **Copyable Patterns**: Code examples and rules should be functional snippets that engineers can immediately adapt for their mono-repos.
- **Progressive Disclosure**: Keep underlying infrastructure clean. Use `.mdc` routing and symlink patterns so the root directory remains uncluttered.
